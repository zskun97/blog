    抽象工厂

    # 抽象工厂

    抽象太难理解 啊阿啊阿啊阿啊

    各方博文中有的时候定义相近，但是示例代码中和定义对不上。。。

    这里根据百度百科中的举例和廖雪峰博客和菜鸟中的举例进行理解

    # 定义

    抽象工厂模式（Abstract Factory Pattern）是围绕一个超级工厂创建其他工厂。该超级工厂又称为其他工厂的工厂

    提供一个创建一系列相关或相互依赖对象的接口。

    当一个产品族中的多个对象被设计成一起工作时，它能保证客户端始终只使用同一个产品族中的对象

    与工厂模式相比, 抽象工厂更为复杂一些… 下面举例是个人觉得比较好理解的

    通俗理解

    以吃早餐举例, 假设我们早餐有两个配置 : 一个主食(包子,油条), 一个喝的(豆浆,豆腐脑) 那么主食有不同具体实现对吧,所以我们用工厂模式获取要吃哪个; 喝的也有不同的实现, 所以也搞一个工厂 这样我们通过这两个工厂就搭配出了一顿早饭, but这是一家早餐店的情景, 如果旁边还有一家早餐店, 提供主食(肉饼,煎饼) 喝的(豆浆,小米粥) 这个情景下, 代码怎么实现呢? (这里有个前提就是早饭只能在一家店吃, 别杠 规则为大)

    两个早餐店相当于两个实现, 所以针对早餐店我们再做一个工厂 这里也不是必须加一个早餐店接口, 加一个工厂; 我们可以将接口和工厂类合起来, 写一个抽象父类, 父类中只有一个实现方法就是获取子类, 同时这个方法不允许子类重写,其他都是抽象方法

    这里举例在代码上可能有一点极端，只是便于理解；场景可能不合适，但意思应该是这样的，第一层工厂按系列分，里面工厂根据产品分

    因为代码使用的通过注解注入，所以使用抽象类get实现子类不中，所以通过另一个工厂来获取。。。就是两层工厂模式

    # 实现

    代码中一堆工厂，一堆实现，我已经晕了。。。太多了 不贴代码了，有兴趣可以看a02_abstractFactory包中的代码； 其实就是定义个早餐店接口（包含不同视频类型的工厂，比如主食工厂，饮料工厂），两个店就是两个实现，然后写一个工厂返回对应的早餐店

    早餐店接口

    public interface IBreakfast {
  //这里根据场景，比如如果只有这个系列只有一个实现，可以直接返回对应实现，不用返回其工厂
    public IDrinkFactory getDrink();
    public IStapleFoodFactory getStapleFood();
}

    具体实现

    @Component
public class LLBreakfast implements IBreakfast {
    @Resource    LLDrinkFactory llDrinkFactory;
    @Resource    LLStapleFoodFactory llStapleFoodFactory;
    @Override    public IDrinkFactory getDrink() {
        return llDrinkFactory;
    }
    @Override
    public IStapleFoodFactory getStapleFood() {
        return llStapleFoodFactory;
    }
}

    工厂

    @Component
public class BreakfastFactory {
    @Resource
    LZBreakfast lzBreakfast;
    @Resource
    LLBreakfast llBreakfast;
    private Map<Integer, IBreakfast> map = new ConcurrentHashMap();
    @PostConstruct
    public void init() {
        //这里1 2 不符合规范，可以写到枚举中
        map.put(1, lzBreakfast);
        map.put(2, llBreakfast);
    }
    public IBreakfast get(Integer type){
        return map.get(type);
    }
}

    测试使用

    @Test
void main(){
    //获取早餐店
    IBreakfast breakfast = breakfastFactory.get(1);
    //获取这个早餐店的饮品工厂
    IDrinkFactory drinkFactory = breakfast.getDrink();
    //获取这个早餐点的主食工厂
    IStapleFoodFactory stapleFoodFactory = breakfast.getStapleFood();
    //获取主食
    stapleFoodFactory.get(1).serve();
    //获取饮品
    drinkFactory.get(1).serve();
}
