# linux操作笔记

# 管理用户和用户组

# 创建用户组

groupadd 参数 用户组名-g: 执行新建用户组的gid-r: 创建系统工作组, 系统用户的组ID小于500-K: 覆盖配置文件 "/etc/login.defs"-o: 允许添加组ID号不唯一的工作组

groupadd -g 1000 playgroup
#创建gid为1000的playgroup组

# 修改组

groupmod 参数 用户组名
-g: 为用户组指定新的组标识号
-o: 与-g同时使用,用户组的新GID 可以与系统已有的相同
-n: 用户组改名

groupmod -g 331 -n playgrp playgroup
## 将playgroup改名playgrp  gid改为331

# 删除用户组

groupdel 用户组名

# 创建用户

useradd 参数 用户名
-g 属组
-u 设置uid
-m 创建家目录
-M 没有家目录
-G 指定属于多个组
-s 指定登录shell
-d 指定家目录
-c 注释
-D 改变它的默认属性
-e 指定的日期是账号失效的日期

user player1
#创建player1用户

# 设置用户密码或修改

passwd 参数 用户名
-l: 锁定已经命名的账户名称
-u: 解开账户的锁定状态
-x: 密码使用最大时间
-n: 密码使用最小时间
-d: 删除使用者的密码
-S: 检查指定使用者的密码认证种类
--stdin: 非交互式修改/设置密码, 弊端是操作日志能查密码, 用history -c 干掉

#交互式改动会提醒你输入新密码
passwd 用户名
#非交互式
echo 123345|passwd --stdin player1

# 修改用户

usermod 参数 用户名
-c: 修改用户账号的备注文字
-d: 修改用户登入时的目录
-e: 修改账号的有效期限
-f: 改密码过期多少天后关闭该账号
-g: 修改用户所属的群组
-G: 修改用户所属的附加群组
-I: 修改用户账号名称
-L: 锁定用户密码, 使密码无效
-s: 修改用户登入后所使用的的shell
-u: 修改用户id
-U: 解除密码锁定

# 删除用户

userdel 参数 用户名
常用的选项是-r, 作用是把用户的家目录一起删掉

# 档案权限与目录配置

# 档案的权限表示

如 drwxr-xr-x

第一个字符表示这个档案的类型, d表示目录, -表示档案 ; 其他还有 l b c

后面分别三个一组,以rwx表示读写执行, 如果没有权限就是 - 表示:

  1. 第一组为所有者的权限,
  2. 第二组是同群组的权限
  3. 其他非本群组的权限

# 属性和权限更改

# chgrp : 该表档案所属群组

# chown : 改变档案拥有者

除了改属主也可以改群组, 如果要改目录下所有子目录的话加-R就可以

chown -R appweb:appgrp QLC
## 将QLC文件及其下的所有文件和目录的属主和分组更改为appweb:appgrp

# chmod : 改变档案的权限

r w x 读 写 执行 还可以用 4 2 1数字来表示

chmod -R 770 QLC     #修改QLC文件夹的权限为自己和同组都可读写执行
chmod -R 740 QLC     #同组只能读

另一种方法修改

u : user g : group o : others a : all

image.png

chmod u=rwx,go=rx     #所有者可读写执行, 同组和非同组可以读执行

这种方式适合不知道原先的文件属性时进行修改的情况

# 权限对于文件和文件夹

# 文件:

r : 可读取文件的实际内容

w : 可以编辑,修改文件的内容,但是不能删除该文件

x : 可以执行该文件

# 文件夹:

r : 可以读取文件夹结构列表, 可以查看文件夹下有哪些文件, 不代表能看文件的具体内容

w : 可以修改目录结构的权限, 可以增删文件, 改名移动等等操作

x : 代表用户是否进入该目录成为工作目录的用途

# 文件和目录管理

常见的处理目录的指令:

cd : 变换目录

pwd : 显示当前目录

mkdir : 创建一个新的文件夹

rmdir : 删除一个空的文件夹

# mkdir

mkdir [-mp] 目录名称
-m 配置目录的权限,不用预设权限
-p 递归创建文件夹,创建多层目录

#示例 创建多层目录,权限771
mkdir -m 711 -p /test/test2/test3

# rmdir

删除空的目录

rmdir [-p] 目录名称
-p 将输入的路径上的空目录一起删除

#会将test/test/test删除, 如果他的父级test/test也变成空了那么也会删除, 会根据路径判断是不是为空
rmdir -p test/test/test

rm -r test #就算test下还有文件会一同删除

# cp 复制

cp 参数 源文件 目标文件
-i : 若目标文件已经存在, 在覆盖时会先询问是否进行覆盖
-p : 将文件属性一起复制过去
-r : 递归复制, 用于目录的复制行为

# rm 移除

rm [-fir] 文件或目录
-f 忽略不存在的文件或目录, 不提醒错误信息
-i 互动模式,删除前会询问是否这么做(基本默认会有)
-r 递归删除

# mv 移动

也可以用来改名

mv [-fiu] 源文件 目标文件
-f 强制, 如果目标档案已经存在, 不会询问直接覆盖
-i 如果目标文件已经存在则询问
-u 如果目标已存在, 且源文件更新时才会更新

# 文件搜索

whereis

locate

locate [-ir] keyword
-i 忽略大小写差异
-r 后面可以接正则表达式
locate根据/var/lib/mlocate内的数据库记载,找出用户输入的文件名

find

find [PATH] [option] [action]

PATH可以指定要搜索的目录

与时间相关的选项:
-atime -mtime -ctime 代表比较的时间属性不同
-mtime n : 在n天之前的24小时内被改动过内容的文件
-mtime +n : n天之前(不含n)被改动过内容的文件
-mtime -n : n天内被改动过内容的文件
-newer file : 与存在的文件比较, 比它新的被查出

与使用者或组名有关的参数:
-uid n : 根据/etc/passwd中记录的uid进行查找
-gid n : 根据gid进行查找
-user name : 根据用户名查找
-group name : 根据组名查找
-nouser : 查找档案的所有者不在/etc/passwd的文件,可以查已删用户的文件
-nogroup : 寻找档案的所有群组不在/etc/group 的文件

与档案权限及名称有关的参数:
-name filename : 搜寻文件名为filename的文件
-size [+-]size : 寻找比size还要大或小的文件
-type TYPE : 找档案类型为TYPE的
-perm mode : 搜寻档案权限等于mode的档案, mode是权限属性的表达式

# 压缩和打包

zip [选项] 压缩包名 源文件或源目录    压缩成zip文件

unzip [选项] 压缩包名     -d：指定解压缩位置

#tar命令 不管解压还是压缩都是   包名在前,路径在后, 好像zip也适用....
-c 建立打包档案
-x 解压或解打包文件    结合 -C   指定目录位置解压
-v 在过程中将正在处理的文件名显示出来
-f 后面接要处理的文件名,要放在最后
-z 通过gzip进行压缩/解压, 档案名最好为 *.tar.gz
-t 可以查看打包文件的内容有哪些文件名

#使用tar压缩文件
tar -zcvf test.tar.gz ./test/
#该命令表示压缩当前文件夹下的文件夹test，压缩后缀名为test.tar.gz
#如果不需要压缩成gz，只需要后缀为tar格式的，那么输入如下命令：
tar -cvf test.tar ./test/

#使用tar解压文件   x标识解压,
tar -xzvf test.tar.gz
#该命令表示把后缀为.tar.gz的文件解压到当前文件夹下。
#如果压缩文件的后缀是.tar，没有gz，则使用命令:
tar -xvf test.tar

#预览压缩包下的文件名
tar -ztvf test.tar.gz

#只解压其中一个文件, 在原本的包名后 跟文件名即可
tar -zxvf test.tar.gz test.py
#打包时, 剔除部分文件,将tmp中的test开头和a开头的全剔除
tar -zcvf test.tar.gz --exclude=/tmp/test* --exclude=/tmp/a* /tmp

#一边打包一边解开  指令中 - 理解为缓冲区, 一边写入一边读出
#功能类似cp -r
tar -cvf - /etc | tar -xvf -

# nohup和&后台运行,进程查看及终止

# 1.nohup

用途: 不挂断的运行命令,会一直前端运行, 默认输出到当前目录的nohup.out文件中, 如果当前目录不可写,输出重定向到$home/nohup.out文件中

语法格式

 nohup Command [ Arg … ] [　& ]

参数说明:

command: 要执行的命令

arg: 一些参数,也可以执行输出文件

&: 让命令在后台执行, 终端退出后命令依旧执行

实例: 执行部署的工程

nohup java -jar coupon-0.0.1-SNAPSHOT.jar

# 2.查看后台执行的命令

jobs: 可以查看当前终端后台进行的任务

ps : 可以查看当前所有进程

# 3.关闭当前后台的进程

kill 命令

# 4.前后台进程的切换与控制

fg: 将后台的命令调至前台继续运行

​ 如果后台有多个命令, 可以先用jobs查看jobnum, 然后用 fg %jobnum将命令调至前台

ctrl+z: 将前端执行的程序放到后台,程序会暂停

bg: 将一个在后台暂停的命令, 变成在后台继续运行

​ 如果后台有多个命令, 可以先用jobs查看jobnum, 然后用 bg %jobnum将命令调至前台

ctrl+c: 会终止当前命令
