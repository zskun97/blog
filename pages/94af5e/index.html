    适配器模式

    # 适配器模式

    # 定义

    将一个类的接口转换成客户希望的另外一个接口，使得原本由于接口不兼容而不能一起工作的那些类可以一起工作。

    适配器模式（Adapter Pattern）是作为两个不兼容的接口之间的桥梁。这种类型的设计模式属于结构型模式，它结合了两个独立接口的功能。

    这种模式涉及到一个单一的类，该类负责加入独立的或不兼容的接口功能。举个真实的例子，读卡器是作为内存卡和笔记本之间的适配器。您将内存卡插入读卡器，再将读卡器插入笔记本，这样就可以通过笔记本来读取内存卡。

    场景

    主要用来解决正在用的项目中适配性问题, 或者第三方接口适配问题

    比如之前一直用的A平台短信接口, 现在要求同时接入B平台, 两个平台的入参出参都有出入; 但是我们不能在自己业务中写过多判断, 所以定义适配器接口, 然后分别针对两个平台的接口做具体适配实现; 然后通过工厂模式对外提供两个平台的适配实例进行使用;

    # 实现

    模拟第三方接口

    /**
 * 阿里云短信服务
 * 这里是伪造第三方接口,传参都是对方规定的
 */
@Component
public class AliyunSMS {
    public void send(AliyunSendDto dto){
        System.out.println("[阿里云] 接收人:"+dto.getPhone()+" 正文:"+dto.getContext());
    }
}

    @Data
public class AliyunSendDto {
    //手机号
    Long phone;
    //正文
    String context;
    //类别
    int type;
}

    /**
 * xxx厂商的短信服务
 * 这里是伪造第三方接口,传参都是对方规定的
 */
@Component
public class XxxSMS {
    public void sendSms(XxxDto dto){
        System.out.println("[xxx] 接收人:"+dto.getMobile()+" 正文:"+dto.getText());
    }
}

    @Datapublic class XxxDto {
    //手机号
    Long mobile;
    //正文
    String text;
    //类别
    String type;
}

    短信接口

    /**
 * 短信接口
 */
public interface SmsApi {
    //我们定义发送短信接口就叫send
    void send(SmsDto dto);
}

    /**
 * 我们自己定义发短信都传这几个参数
 */
@Data
public class SmsDto {
    Long phone;
    String Message;
    String type;
}

    适配器实现

    /**
 * 对阿里云的适配, 参数转换,结果转换等等
 */
@Component
public class AliyunAdapter implements SmsAdapter {
    @Resource
    AliyunSMS aliyunSMS;
    @Override
    public void send(SmsDto dto) {
        AliyunSendDto aliyunSendDto = new AliyunSendDto();
        aliyunSendDto.setPhone(dto.getPhone());
        //参数不同处理
        aliyunSendDto.setContext(dto.getMessage());
        aliyunSendDto.setType(Integer.parseInt(dto.getType()));
        aliyunSMS.send(aliyunSendDto);
    }
}

    /**
 * 对xxx平台的适配, 参数转换,结果转换等等
 */
@Component
public class XxxAdapter implements SmsAdapter {
    @Resource
    XxxSMS xxxSMS;
    @Override
    public void send(SmsDto dto) {
        XxxDto xxxDto = new XxxDto();
        xxxDto.setMobile(dto.getPhone());
        xxxDto.setText(dto.getMessage());
        xxxDto.setType(dto.getType());
        xxxSMS.sendSms(xxxDto);
    }
}

    通过工厂获取实例

    @Component
public class SmsFactory {
    @Resource
    AliyunAdapter aliyunAdapter;
    @Resource
    XxxAdapter xxxAdapter;
    private Map<String, SmsAdapter> implMap = new ConcurrentHashMap();
    @PostConstruct
    public void init() {
        implMap.put("aliyun", aliyunAdapter);
        implMap.put("xxx", xxxAdapter);
    }
    public SmsAdapter get(String type){
        return implMap.get(type);
    }
}

    测试调用

    @Resource
SmsFactory smsFactory;
@Testvoid main(){
    //构建消息体
    SmsDto smsDto = new SmsDto();
    smsDto.setPhone(13512341234L);
    smsDto.setMessage("刮风啦下雨啦");
    smsDto.setType("1");
    //同一个消息使用不同的服务商，如果后期需要再接其他服务商，就再添加个适配器实现，然后工厂中配置就好了
    smsFactory.get("aliyun").send(smsDto);
    smsFactory.get("xxx").send(smsDto);
}

    # 总结

    Adapter模式可以将一个A接口转换为B接口，使得新的对象符合B接口规范。

    编写Adapter实际上就是编写一个实现了B接口，并且内部持有A接口的类：

    在Adapter内部将B接口的调用“转换”为对A接口的调用。

    过多地使用适配器，会让系统非常零乱，不易整体进行把握； 如果内部接口不如重构，第三方接口无法改动但是还要一起用时可以考虑
