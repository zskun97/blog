# Zookeeper

# 概述

官方文档上这么解释zookeeper，它是一个分布式服务框架，是Apache Hadoop 的一个子项目，它主要是用来解决分布式应用中经常遇到的一些数据管理问题，如：统一命名服务、状态同步服务、集群管理、分布式应用配置项的管理等。

是一种为分布式应用所设计的高可用、高性能且一致的开源协调服务,它提供了一项基本服务:分布式锁服务. 后来有摸索出其他使用方法: 配置维护、组服务、分布式消息队列、分布式通知/协调等

上面的解释有点抽象，简单来说zookeeper=文件系统+监听通知机制。

常用操作:

create [-s] [-e] path data    创建节点 存储的数据       -s指定是序列化顺序节点    -e指定是临时节点
临时节点会话结束后自动删除, 不允许有子节点

ls path     查看路径下有哪些子节点

get [-s] [-w] path       获取该节点数据       -s获取详细节点信息   -w获取存储的数据

delete [-v version] path  删除节点,如果有子节点则不行

deleteall path 递归删除

set [-s] [-v version] path data   存储数据   -s存储后展示节点信息   -v指定版本修改,不符合提醒version No is not valid :

get -s path

zsk                 //存储的数据
cZxid = 0x5         //节点被创建的zxid
ctime = Wed Jun 16 07:11:27 UTC 2021            //节点创建的时间
mZxid = 0x5         //节点被修改的zxid
mtime = Wed Jun 16 07:11:27 UTC 2021            //节点被修改的时间
pZxid = 0x5          //最新修改的zxid,和mzxid一样的吧
cversion = 0          //所拥有的的子节点被修改的版本号
dataVersion = 0       //数据版本
aclVersion = 0              //
ephemeralOwner = 0x0      //如果是临时节点,值便是节点拥有者的会话id,否则为0
dataLength = 3             //节点数据长度
numChildren = 0               //子节点数量
