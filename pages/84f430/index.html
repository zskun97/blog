    jvm调优和linux

    # jvm调优和linux

    # 怎么排查生产服务器变慢

    1. top

      使用top命令看整机的情况

      uptime 精简版系统性能

    2. cpu

      使用vmstat命令查看

      其他命令:

      查看所有核信息 mpstat -P ALL 2

      每个进程使用cpu的用量分解信息 pidstat -u 1 -p 进程编号

      例: vmstat -n 2 3 没两秒刷新一次,刷新三次

    3. 内存

      使用free -m 命令

      其他命令 pidstat -p 进程号 -r 采样间隔秒数

    4. 硬盘

      使用df -h命令

    5. 磁盘IO

      使用iostat命令

      其他命令 pidstat -d 采样间隔秒数 -p 进程号

      例: iostat -xdk 2 3

    6. 网络IO

      使用ifstat命令

      默认本地没有 需要下载ifstat

      例: ifstat 采样间隔秒数

    # 如果生产出现cpu占用过高, 怎么分析和解决

    结合linux和jdk命令一起分析

    1. 使用top命令找出占用cpu高的 pid

    2. ps -ef或者jps进一步定位, 得知是什么后台程序

    3. 定位具体的线程或者代码

      ps -mp 进程 -o THREAD,tid,time

      -m : 显示所有的线程

      -p : pid进程使用cpu的时间

      -o : 改参数后是用户自定义格式

    4. 将需要的线程id转换为16进制

      printf "%x\n” 线程id

    5. jstack 进程id | grep tid(16进制线程id) -A60 //打印前60行
