    ConfigurableApplicationContext run = SpringApplication.run(SpringBootDemoApplication.class, args);

    # 自动装配原理

    默认扫描当前包及其子包里面所有的组件

    依赖包中的META-INF/spring.factories配置了默认需要导入的全类名

    按需加载所有自动配置项

    springboot所有自动配置功能都在spring-boot-autoconfigure包里面,这个包里面有全场景的自动配置类

    配置类上有各种condition判断条件是否需要注册bean

    # 请求映射原理

    1. 从dispatcherServlet的doDispatch开始

    2. doDispatch方法中调用getHandler()获取请求对应的Handler(Controller方法)处理

    3. getHandler方法中遍历处理器映射器handlerMappings RequestMappingHandlerMapping：保存了所有@RequestMapping 和handler的映射规则

    4. AbstractHandlerMethodMapping.lookupHandlerMethod方法, 从mapping中根据路径先查找一个集合, 然后从集合中根据请求方式进行匹配,如果最终匹配到多个会报错
